| # | Diagnostic | Script | Finding |
|---|---|---|---|

| D2 | Class structure (GMM, t-SNE, NN counts) | `scripts/investigate_class_structure.py` | 62% of L2 samples have an L1 sample as nearest neighbor in CNN-BiLSTM emb. L4 sits 0.6+ from everyone (easy outlier). |

## Key structural facts uncovered

- Train and test users are **disjoint** (`User_001..060` vs `User_061..100`). User identity cannot be a feature.
- Data is **1-Hz aggregation** (mean + std per second of 6 axes). Anything faster than 0.5 Hz is aliased away.
- **L1 and L2 are not separable** in any feature space we built. The 62% NN-is-L1 rate for L2 samples is a property of the data, not the models.
- **OOF→LB inversion is large** (+0.025 to +0.028 typical), suggesting train is harder than test. This is the user-shift signature — train users have more variability than test users.
- The OOF→LB inversion **depends on thresholds**, not just the model. Identical OOF (0.7880) can map to LB 0.7991, 0.8154, or 0.7698 depending on threshold choice.
- **Gravity features encode user identity**. EO confirmed: all 11 gravity columns dropped during feature selection.
- A faithful **CNN-BiGRU + Evidential Alignment** re-attempt (2026-05-30, GPU server) reconfirmed the ceiling: every GRU variant — EA (LB 0.7187), plain (0.7105), per-file-norm (0.7233) — is a strong standalone sequence model but **below the 0.8154 ensemble and adds nothing to the stack**. EA was *neutral* (≈ plain BiGRU), not the bottleneck; the **user-shift** is. The one real lever was **per-file (cross-subject) normalization**, which made the GRU user-robust (in-dist≈OOF) and rescued L4 — but it could not move **L2** standalone, which stays ~0.23 for all sequence models.
- **Nuance (later finding, 2026-05-30):** the GRU adds *zero* value in a **linear** simplex blend (weight ~0.05, no gain), but in a **nonlinear LGBM boosting stacker** it contributes a small, *reproducible* signal that moves **L2**: +0.024–0.025 macro-L2 and +0.008–0.015 macro across **two** independent stacker tests (`scripts/_boost_stack_gru.py`, and combo-recipe with/without GRU). This is the first thing in the GRU/EA arc to move L2 at all. **Final production-blend test (frozen OOFs, `scripts/blend_v6_plus_gru.py`):** the 0.8154 pipeline rebuilds *exactly* from frozen OOFs — mixing the GRU at weight β=0 reproduced the production 0.7880 OOF to the digit (so the combo-*training* reproduction, blocked by env drift, was never needed for this). Sweeping β: best β=0.10 → 0.7883 (**+0.0003, noise**) and L2 *drops* 0.384→0.376. The GRU's L2 signal is **redundant with and weaker than the hierarchical P2's** (P2 alone L2=0.384 vs GRU ~0.23), so blending it in dilutes the stronger signal. **Definitive: the GRU does not improve the production ensemble across any integration path (standalone, linear blend, boosting stack, production blend).** **EA recalibration of P2** (`scripts/ea_recalibrate_p2.py`) — Dirichlet head on P2's log-probs + Stage-2 uncertainty-weighted calibration on held-out users, re-blended. *Apparent* OOF gain 0.7880→0.7924 (+0.0044) **but it was a threshold-overfit artifact**: the re-tuned thresholds scored **LB 0.7990** (< 0.8154), and under the *frozen* top-1 (0.8154) thresholds EA-P2 = **0.7845 < plain-P2's 0.7880**. A textbook **OOF→LB threshold trap** — the +0.0044 came from re-optimizing the L1/L2 grid on the EA OOF, not from EA helping. **EA-P2 is a negative.** (Lesson reinforced: when comparing any recalibration, hold the threshold config fixed — re-tuning thresholds manufactures OOF gains that don't transfer.) Nothing in the EA/GRU arc beats the 0.8154 production config.
- **Final attempt — dedicated binary CNN-BiGRU + EA L1↔L2 specialist** (`src/models/train_l1l2_ea_specialist.py`, walking subset only): OOF L1↔L2 macro-F1 **0.5621** (L2 recall 0.550) — *worse* than the existing 6-class neurals (gru 0.608, cnn 0.620) and combo (0.677). The binary walking subset (~280 L2/fold) starves it of data, while the 6-class models learn better L1↔L2 reps from the full set. A *feasibility check* first confirmed the neurals carry **complementary L2 recall** (0.56–0.63 vs the GBDT stack's 0.11–0.28) — a real signal — **but it cannot be converted into a production gain**: GRU-in-blend (flat), EA-P2 (threshold trap), and a dedicated specialist (worse) all fail to beat the production L2=0.384. **Definitive close: the neural/EA/GRU direction is fully exhausted; the L2 bottleneck is structural; 0.8154 stands at the data/toolset ceiling.**

A tuned `combo_full_v2 + gru_pn_ms` run (`--name combo_full_v3gru`) tested whether the gain survives into the *production* stacker. **Resolved: it did not** — tuned combo+GRU came in at OOF **0.7291 / L2 0.231**, below the production `combo_full_v2` (0.7733 / L2≈0.31). The matched-param L2 gain is real but does not lift a properly-tuned stacker. (Caveat: the 15-trial Optuna repro underperformed the original combo_full_v2 tuning, so the GRU effect at production tuning is only bounded — but it's a clear miss vs the 0.7733 bar regardless.) **GRU/EA arc conclusively closed; primary 0.8154 / backup 0.8114 hold.**

## DG augmentation arc (reopened 2026-05-31): conditional subject-invariant representation learning

Motivated by a fresh literature review (GILE/AAAI'21, DDLearn/KDD'23, CCIL/AAAI'25) and the diagnosis that the problem is *learn L2/L3/L5 characteristics while removing the user-spurious attribute* — which boosting on aggregate features cannot do, only representation learning can. The earlier failures (SICL, DANN) used **marginal** subject-suppression and killed anatomy-correlated activity signal; the fix is **conditional** invariance (invariant to user *within* each class).

|---|---|---|---|
| 20 | **CISC** — CNN-BiGRU + class-weighted CE + λ·**conditional cross-user SupCon** (pulls same-class-*different-user* pairs together) | `src/models/train_dg_cisc.py`, `src/utils/cond_supcon.py` | Standalone OOF macro **0.6396**. **L2 recall 0.458** (prod 0.372, best GRU 0.23) — cross-user SupCon genuinely lifts cross-subject L2 *recall*; but L2 *precision* crashes to 0.106 (over-predicts L2). |
| D7 | Feasibility/complementarity gate | `scripts/dg_feasibility.py` | **PASSED.** DG rescues **84 of production's 225 L2 misses (37.3%)**, net-positive on L2 (DG 84 vs prod 53). **Oracle ceiling (ideal per-sample gate) = 0.8414** vs prod 0.7880. L3/L5: production dominates (net-negative for DG). |
| 21 | Gated integration under **FROZEN top-1 thresholds**, nested-CV honest | `scripts/dg_gated_integrate.py` | **L2-only calibrated injection w=0.05 → 0.7880→0.7888 (+0.0008), nested-validated** (w=0.05 chosen in all 5 user-folds → transfers). **First methodologically-honest positive in the whole neural arc.** Injecting L2+L3+L5 → nested picks w=0 (L3/L5 net-negative). |
| 22 | Confidence-gated L1→L2 hard rescue | `scripts/dg_gated_rescue.py` | Nested **−0.0088**. | ref | LogisticRegression stacker over [prod, DG] + re-tuned thresholds | (in `dg_gated_integrate.py`) | OOF 0.7796 (< production) — naive stacking still fails even with optimistic re-tuned thresholds. |

**v1 verdict:** a **real but tiny honest gain (+0.0008 OOF, nested-validated, frozen thresholds)** — `sub_dg_cisc_v1_gated_w05.csv`. The conditional cross-user SupCon *does* produce complementary L2 signal that survives integration (unlike everything before it), but the capture is **precision-limited**: DG over-predicts L2, and its confidence can't isolate the 84 good rescues from the oracle's 0.8414 headroom. Root cause is the same 1 Hz information ceiling.

| 23 | **v2 — 3-seed multi-seed, milder sqrt oversampling** (`--sampler-power 0.5`, seeds 42/7/23, averaged → `dg_cisc_v2ms`) | `scripts/run_dg_v2_queue.sh`, `scripts/avg_dg_seeds.py` | Better standalone: OOF **0.6817** (L2 prec up, L1 recall 0.589→0.778). **More** complementary signal: rescues **91/225 L2 misses (40.4%)**, oracle ceiling **0.8457**, DG L2-recall 0.707. **But frozen-threshold injection captured LESS: nested +0.0002** (vs v1's +0.0008). `sub_dg_cisc_v2ms_gated_w05.csv`. |

**DG-arc definitive verdict (2026-05-31).** Conditional cross-user SupCon is the **first** method in the entire neural/EA/GRU arc to produce complementary L2 signal that *survives* frozen-threshold integration (validating the diagnosis: *conditional* within-class subject-invariance works where SICL/DANN's *marginal* suppression failed). **But every realizable integration yields only +0.0002–0.0008 OOF — within LB noise** — and, tellingly, the *better* v2 model captured *less* than v1. The oracle's 0.84+ per-sample ceiling is **unreachable**: no available gate signal (DG confidence, LR stacker, hard L1→L2 rescue) can isolate the ~90 genuinely-rescuable L2 from DG's many false positives, because DG's L2 confidence is itself unreliable (1 Hz erased the L1/L2 gait signal). **The binding constraint is the data, not the model or the loss.** Best artifact: `sub_dg_cisc_v1_gated_w05.csv` (+0.0008 OOF, frozen thresholds, nested-validated across users). Primary **0.8154** / backup **0.8114** still stand as the production submissions.

**Empirical LB (submitted 2026-05-31): both `sub_dg_cisc_v1_gated_w05` and `sub_dg_cisc_v2ms_gated_w05` scored exactly 0.8154 — an exact tie with production.** The +0.0008 OOF was within noise as predicted; the ~2 flipped L2 predictions netted zero on the public split. This is the clean empirical confirmation that **0.8154 is the data-bound ceiling**: the strongest principled method we could build — conditional subject-invariant representation learning, properly integrated under frozen thresholds — lands *exactly* on it, neither helping nor hurting. (Note: the frozen-threshold + nested-CV discipline correctly predicted a flat LB; no threshold-trap mirage, no regression.)

## Feature-engineering arc (2026-05-31): pseudo-gyroscope from the gravity trajectory

Dataset provenance (Bruno et al. 2014, **wrist-worn** ADL; **gravity NOT removed**) → the per-second `mean_x/y/z` trace the wrist **orientation trajectory**; its time-derivative approximates the gyroscope the HAR literature says is needed to separate confusable activities (and which we lack as a sensor). We previously used orientation only *statically* (window-mean angles + coarse 5-chunk drift). Built **36 orientation-DYNAMICS features** (`scripts/orient_pseudogyro_model.py`): pseudo-angular-speed (Δ between consecutive gravity unit-vectors), cumulative rotation, tilt-vs-window-mean trajectory (mounting-invariant), pitch/roll reversals, gravity-magnitude bursts, intensity (std-channel) dynamics + coefficient of variation, and rotation↔intensity coupling. Convention-invariant core (robust without the exact axis frame).

**Feature-engineering verdict.** The pseudo-gyroscope idea was *correct* — orientation dynamics reconstructed from the gravity trajectory carry real L1↔L2 signal (the first feature family grounded in the dataset's actual physics, not generic HAR practice). But it lands at the **same realizable ceiling**: gated L2-injection gives +0.0005 OOF (within LB noise — the equivalent DG injection empirically tied 0.8154), and it does **not stack** with the DG signal. Both tap a single small pool of recoverable L2 complementary signal (~+0.0005–0.0008 OOF). The features' fuller value would require adding them *inside* the combo / Fine_walk stacker — blocked by the env-drift reproduction issue. **Three independent principled methods — neural conditional-invariance (DG), GBDT pseudo-gyro features, and their combination — all converge on the same sub-LB-resolution L2 gain. This is the firmest triangulation yet that 0.8154 is the data-bound ceiling.**

### ⚠️ LB UPDATE / CORRECTION (submitted 2026-05-31): orientation injection BEAT production

**`sub_dg_cisc_orient_lgbm_gated_w15.csv` scored 0.8184 on the public LB — +0.0030 over 0.8154, the FIRST real improvement in the whole project.** This **overturns the inferred "ties 0.8154"** above. What happened: the +0.0005 nested-OOF L2-injection edge (orient_lgbm, w=0.15 chosen by nested CV) **amplified to +0.0030 on the public split** — a handful of correct L1→L2 flips on the *rare* L2 class, scored on a smaller public subset, move macro-F1 disproportionately. The effect is **nested-CV-validated** (real, transfers across train users), but the +0.0030 *magnitude* exceeds the OOF estimate, so part of it is **public-split favorability → private-LB hold is uncertain** (expect neutral-to-+0.003). DG `v1`/`v2ms` (w=0.05) scored 0.8154; orient's w=0.15 injects *more* L2 mass and its specific flips were public-correct. **Lesson reasserted (the project's own): SUBMIT, don't infer — the OOF→LB map is favorably noisy here, and my "it'll tie" inference was wrong.** Pushing further via a *stronger* nested-validated L2 source (rich-stack 3-seed 6-class LGBM, `scripts/orient_strong_source.py`); **w stays nested-chosen, NOT tuned to the public LB** (that would be the threshold-trap in disguise). Current best submission: **0.8184**; recommend hedging finals with a robust pick for the private split.
